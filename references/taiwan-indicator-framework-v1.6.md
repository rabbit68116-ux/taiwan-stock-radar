# Taiwan Indicator Framework v1.6

Version: `v1.6`

這份文件整理 Taiwan Stock Radar `v1.6` 的單股指標框架。

`v1.6` 的重點不是新增更多指標名稱，而是把台股最實用的欄位整理成一套 **能和 08:30 日盤簡報自然銜接** 的單股研究系統。

## 1. 設計原則

`v1.6` 對台股單股研究的基本要求是：

- 先有日盤環境，再做單股判讀
- 不依賴單一技術指標
- 不把新聞標題當成決策本體
- 技術面、量價、籌碼、基本面、事件面必須一起看

## 2. 六層訊號引擎

### A. 趨勢引擎

核心欄位：

- `5MA / 10MA / 20MA / 60MA / 120MA / 240MA`
- 是否站上 `20MA / 60MA`
- `20MA` 與 `60MA` 斜率
- `52W high / low distance`

用途：

- 定義多頭、整理、轉弱
- 決定要找突破、拉回，還是先避開

### B. 動能引擎

核心欄位：

- `MACD`
- `RSI(14)`
- `KD(9,3,3)`
- 近 `5 / 20` 日報酬
- 布林通道位置

用途：

- 判斷短中線強弱
- 判斷過熱、過弱與節奏轉折

### C. 量價引擎

核心欄位：

- 當日成交量
- `5日 / 20日` 均量
- 量比
- 是否放量突破
- 是否回檔縮量
- 價漲量增 / 價跌量增

用途：

- 避免假突破
- 確認主升段與承接品質

### D. 籌碼引擎

核心欄位：

- 外資買賣超
- 投信買賣超
- 自營商方向
- 融資增減
- 融券增減
- 借券賣出 / 借券餘額
- 連買 / 連賣天數

用途：

- 補足台股特有的部位結構與法人行為
- 確認股價上漲是否有健康資金支持

### E. 基本面引擎

核心欄位：

- 月營收 `MoM / YoY`
- `EPS`
- `ROE`
- 毛利率
- 營益率
- 現金流
- `PE`
- 殖利率
- `PB`

用途：

- 確認股價強勢是否有商業品質支撐
- 幫助區分高品質領導股與短線題材股

### F. 事件引擎

核心欄位：

- 法說會日期
- 財報公布日
- 月營收公布日
- 除權息日期
- 重大訊息
- 產業題材
- ADR / 費半 / 美股連動

用途：

- 管理時間點風險
- 避免 setup 被事件直接打掉

## 3. 交易風格切換

### A. short_term

優先看：

- `5MA / 10MA / 20MA`
- 成交量
- `KD`
- `RSI`
- 短期法人買賣超

### B. swing

優先看：

- `20MA / 60MA / 120MA`
- `MACD`
- 放量突破 / 回檔縮量
- 投信 / 外資趨勢
- 月營收與財報摘要

### C. position

優先看：

- 殖利率
- 配息穩定性
- `PE`
- `ROE`
- 現金流
- 產業趨勢
- 大盤位置

## 4. 最實用的策略組合

### A. 趨勢突破

- `20MA` 上彎
- 股價站上 `60MA`
- 放量突破前高
- `MACD` 翻正
- 外資或投信同步買超

### B. 回檔承接

- 多頭排列
- 回測 `20MA` 或 `60MA`
- 回檔量縮
- `KD` 低檔黃金交叉
- 法人沒有明顯轉賣

### C. 價值加技術

- `PE` 合理
- 殖利率具吸引力
- 月營收 `YoY` 轉強
- 股價重新站回 `60MA`
- 量能開始放大

## 5. 與 08:30 日盤簡報的關係

`v1.6` 的單股判讀不應脫離日盤簡報。

日盤簡報主要決定：

- 今天整體是偏積極、偏中性，還是偏防守
- 哪些族群應優先觀察
- 哪些風險要提高折扣

六層訊號引擎則負責回答：

- 這檔股票是否真的值得參與
- 該用哪種交易風格理解
- 買賣點是否足夠清楚

## 6. 核心原則

不建議單獨使用：

- 只有 `KD`
- 只有 `RSI`
- 只有本益比
- 只有新聞標題

真正可用的台股判讀，必須把：

- 趨勢
- 量價
- 籌碼
- 基本面
- 事件

一起整合進同一份單股研究文件。
