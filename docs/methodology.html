Taiwan Stock Radar v1.7 如何運作 How Taiwan Stock Radar v1.7 operates

`v1.7` 的方法論重點是：分析邏輯只存在於 shared core，交付面可以是 skill、CLI 或 Windows desktop。這樣做的目的，是讓每日報告與單股委員會在任何介面都能維持同一份結論結構。

The key methodological rule in `v1.7` is that analysis logic exists only in the shared core, while delivery surfaces can be the skill, the CLI, or the Windows desktop app. This keeps the daily brief and the stock committee structurally consistent in every interface.

1. 開市前環境層1. Premarket environment

系統先讀台股夜盤、美股大盤、半導體與 VIX，建立台股開盤前的風險偏好基底。The system first reads Taiwan night-session, US broad indices, semiconductors, and VIX to build the overnight risk foundation before Taiwan opens.

2. 08:30 日盤報告2. The 08:30 market brief

接著整合 Yahoo奇摩股市與 Anue 鉅亨網，形成 10 大訊息、開盤偏向、整體分數、族群觀察與風險提示。It then integrates Yahoo Taiwan and Anue pages into ten key messages, an opening bias, an overall score, sector watchlists, and risk flags.

3. 單股委員會3. Single-stock committee

當使用者指定股票時，系統會先確保同日 daily brief 已存在，再啟動六層訊號引擎、八位專家 AI 分析師與 style profile。When a stock is requested, the system first ensures the same-day daily brief exists, then activates the six signal engines, eight specialist AI analysts, and the selected style profile.

4. 策略與驗證4. Strategy and validation

主策略、次策略與 avoid strategy 會由 Strategy Architect 提名，再由 Validation layer 檢查 performance、tail risk、robustness 與 execution clarity。The Strategy Architect nominates the primary, secondary, and avoid strategies, after which the validation layer checks performance, tail risk, robustness, and execution clarity.

5. 輸出與 archive5. Outputs and archive

所有 workflow 都同時寫入 latest output 與 archive output，讓 CLI、文件站與 desktop app 都能讀到同一份歷史。Every workflow writes both a latest output and an archive output so the CLI, docs site, and desktop app can read the same history.

6. Windows desktop 的角色6. The role of the Windows desktop app

Windows desktop 只做 GUI 層，不複製 business logic。這是 `v1.7` 能夠雙軌同步更新的關鍵。The Windows desktop app is only a GUI layer and does not duplicate business logic. That is the key to keeping the two tracks synchronized in `v1.7`.
