Taiwan Stock Radar 如何執行單股指標研究會議 How Taiwan Stock Radar runs a single-stock indicator committee

這一頁說明 Taiwan Stock Radar `v1.5` 的核心分析序列。流程先從台股開市前環境評估開始，再進入個案定義、訊號引擎校準、風格切換、專家質詢與驗證，最後才形成操作建議。

This page outlines the analytical sequence behind Taiwan Stock Radar `v1.5`. The workflow starts with the Taiwan premarket environment, then moves through case definition, signal calibration, style switching, specialist review, and validation before producing an action plan.

1. 開市前環境評估 1. Premarket environment assessment

如果分析時間點在台股開盤前，系統會先讀取台股夜盤、S&P 500、Nasdaq、SOX、TSM ADR、NVIDIA 與 VIX，形成一份開盤偏向、族群觀察與風險提示。這一步的目的是避免單股判讀脫離隔夜市況。

If the task happens before the Taiwan open, the system first reads Taiwan night-session, the S&P 500, Nasdaq, SOX, TSM ADR, NVIDIA, and VIX to form an opening-bias brief, sector watchlist, and risk flags. This keeps the stock analysis grounded in the overnight environment.

2. 個案設定 2. Case setup

流程接著定義一檔股票、一個分析日期、一個研究週期與一個明確問題。這能避免討論失焦，也避免 agent 用模糊問題產生模糊答案。

The workflow then defines one stock, one analysis date, one time horizon, and one explicit research question. That keeps the meeting focused and prevents vague answers.

3. 台股訊號引擎校準 3. Signal-engine calibration

在下方向判斷之前，系統會先看趨勢、動能、量價、籌碼、基本面與事件六層訊號。單一股票不能只靠一個圖形或一則新聞來理解。

Before any directional call is made, the system reviews trend, momentum, price-volume, chip flow, fundamentals, and events. A stock should not be judged from only one chart pattern or one headline.

4. 交易風格切換 4. Style-profile switching

`v1.5` 不會假設短線、波段與中長線的判讀標準相同。系統會先決定 active style，再調整各引擎的權重，也會讓開市前環境影響早盤積極度。

`v1.5` does not assume that short-term, swing, and position trading should read a stock the same way. The system selects the active style first, then adjusts engine weights and early-session aggressiveness using the premarket tone.

5. Specialist 獨立簡報 5. Independent specialist briefs

技術面、籌碼面、基本面、催化因子、風控、策略選擇與驗證角色會先各自形成初步看法。這能避免第一個強勢敘事過早主導整場會議。

Technical, chip-flow, fundamental, catalyst, risk, strategy-selection, and validation specialists each produce an initial view independently so that the first strong narrative does not dominate the whole meeting.

6. 策略提名 6. Strategy nomination

Strategy Architect 會先判斷這檔股票適合用哪種交易策略來理解，例如趨勢跟隨、突破、拉回承接，還是根本不該採用短線反彈思維。

The Strategy Architect decides which trading strategy best fits the stock, such as trend-following, breakout, pullback continuation, or whether mean-reversion logic should be avoided entirely.

7. 驗證面板 7. Validation panel

Quant Validation Analyst 會用績效、尾部風險、交易品質與穩健度指標檢查這個個案是否足以支撐操作。這一步的目標，是防止會議只剩下漂亮的故事。

The Quant Validation Analyst tests whether the setup deserves action by reviewing performance, tail-risk, trade quality, and robustness metrics. This step exists to prevent the meeting from becoming story-driven.

8. 交叉質詢與權重整合 8. Cross-examination and weighted synthesis

主持 agent 會在開市前環境、訊號引擎檢視、風格適配、策略適配與驗證結果都進場後，才整合各專家角色的意見。最終結論不是投票平均，而是依角色權重與風險反對意見共同形成。

The chair agent synthesizes the specialists only after the premarket environment, signal-engine review, style fit, strategy fit, and validation are on the table. The final verdict is not a simple vote, but a weighted conclusion shaped by both conviction and visible objections.

9. 決策文件輸出 9. Decision packet

最後輸出會包含開市前環境摘要、signal-engine summary、style profile、策略模組選擇、validation panel、base/bull/bear 三情境、建議買入區、進場條件、停損、停利階梯、失效條件，以及保留下來的 dissent memo。

The final packet includes the premarket environment summary, signal-engine summary, style profile, strategy-module selection, validation panel, base/bull/bear scenarios, preferred buy zone, entry triggers, stop-loss, take-profit ladder, invalidation, and preserved dissent notes.
